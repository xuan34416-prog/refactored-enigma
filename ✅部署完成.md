## 📋 发布新版本的流程

### 小更新（Web 资源）

```bash
# 1. 修改代码
# 2. 更新 www/sw.js 中的 CACHE_NAME
# 3. git add . && git commit -m "更新说明" && git push
```

### 大更新（需要 APK）

```bash
# 1. 更新版本号（3 个地方）
#    - package.json: "version": "1.2.0"
#    - android/app/build.gradle: versionCode 3, versionName "1.2.0"
#    - www/update-check.js: currentVersion: '1.2.0'

# 2. 构建 APK
cd android && ./gradlew assembleRelease

# 3. 创建 Release 并上传 APK
#    GitHub → Releases → Create a new release

# 4. 更新 www/version.json
#    修改 version 和 changelog

# 5. 提交并推送
git add . && git commit -m "release: v1.2.0" && git push
```

## ⚠️ 重要提醒

### 数据保留
- ✅ **覆盖安装**：数据会保留
- ❌ **卸载重装**：数据会丢失

### 签名密钥
- 密钥位置：`my_keys/tomato-key.jks`
- 务必保管好密钥文件
- 每次发布必须使用相同的密钥签名

### 版本号管理
每次发布更新时，记得更新以下文件：
1. `package.json` - `"version": "1.2.0"`
2. `android/app/build.gradle` - `versionCode`, `versionName`
3. `www/update-check.js` - `currentVersion`
4. `www/version.json` - `"version"`
